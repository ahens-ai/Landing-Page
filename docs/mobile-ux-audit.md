# ahens.ai Mobile UX Audit

| Priority | Issue found | Affected component/page | Mobile impact | Fix applied in this pass |
| --- | --- | --- | --- | --- |
| High | Mobile navigation existed, but hidden drawer links could remain in the DOM and the drawer did not trap focus. | `components/SiteHeader.tsx`, global header CSS | Keyboard users could tab into hidden navigation; phone users did not get a full-screen product-grade drawer. | Convert the mobile menu into a mounted dialog with Escape close, focus return, focus cycling, safe-area spacing, larger controls, utility links, and accordion sections. |
| High | The mobile hero inherited desktop-scale layout and visual density. | `app/page.tsx`, `components/HeroOrbit.tsx`, hero CSS | Critical message and CTAs could be pushed below the first viewport; animated visuals were too tall for 320-430px screens. | Add mobile-specific hero sizing, full-width CTAs, reduced visual height, lighter node display, and safer dynamic viewport sizing. |
| High | Page gutters were hard-coded around `calc(100% - 32px)` at small widths. | `app/globals.css`, `.container` | Small screens had inconsistent breathing room and higher risk of edge collisions. | Add mobile gutter variables for 320px, 360px, 480px, tablet, and desktop widths; use them in `.container`. |
| High | Forms used two-column desktop grouping and a longer name split on mobile. | `components/ContactForm.tsx`, contact CSS | More fields and narrower inputs made phone completion slower. | Simplify contact form to one name field, work email, company, optional exploration note; add mobile-friendly input attributes and full-width mobile submit. |
| Medium | Architecture and card sections depended on wide desktop layouts and hover polish. | `components/ArchitectureMap.tsx`, architecture/capability/solution CSS | Diagrams and multi-column cards felt like compressed desktop and created long dense scrolling. | Guard pointer hover to mouse, create tap-first mobile architecture controls, and use contained snap rails for dense card groups with visible affordance. |
| Medium | Several touch targets were under or near the 44px minimum. | Header, links, CTAs, footer, card CTAs | Tapping was less comfortable one-handed, especially on 320-375px screens. | Enforce 48px mobile buttons/menu controls and larger mobile link/card tap zones. |
| Medium | Heavy backdrop/filter/continuous animation layers remained active on phones. | Header/menu, hero plane, visuals, body texture | Mobile GPU/CPU work was higher than needed while scrolling. | Reduce mobile backdrop filters, hide non-essential hero nodes, pause/simplify motion through mobile and reduced-motion rules. |
| Medium | Newsletter and footer forms could crowd at narrow widths. | `NewsletterForm`, footer/newsletter CSS | Input/button groups could feel cramped or overflow-prone. | Use one-column mobile form layout, larger controls, safer wrapping, and container-aware widths. |
| Low | Placeholder copy appears in several page sections. | Home, platform, solutions, insights, contact, company | The mobile experience is usable but still carries launch-placeholder messaging. | Preserve placeholders as product/content scope, but remove the most visible footer placeholder wording and list remaining placeholders as known limitations. |
